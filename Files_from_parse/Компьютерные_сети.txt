/&*/---Компьютерные сети---/&*/Технология Ethernet/&*/Виды связей в компьютерных сетях /&*/

Simplex - односторонняя связь. Один источник вещает, остальные получают
Примеры:
- теле, радио вещание
- спутники GPS

Half-duplex - двусторонняя связь (одновременно может передавать только один из абонентов)
Второй при этом может только слушать
Примеры:
-общение по радио // недостаток - возникающие коллизии, когда вещать пытаются сразу оба абонента

Full duplex или просто duplex 
Пример:
- разговор по телефону  // коллизии не возникают

/&*/---Компьютерные сети---/&*/Технология Ethernet/&*/ Модель OSI и TCP/IP /&*/

OSI   - Open System Interconnection  //взаимодействие открытых систем
Придумано 7 уровней, на основе которых должно было строиться взаимодействие в компьютерных сетях и осуществляться построение оборудования
1) Физический уровень (описывают физические параметры как должны связываться устройства (количество каналов, перепад напряжения...)
2) Канальный уровень (порождает понятие пакета (от кого к кому) и что передается) по определённому стандарту
3) Сетей уровень (ограниченное broadcast вещание) Internet протокол, позволяет связывать между собой эти глобальноограниченные сети
4) Транспортный уровень (позволяет определить какому приложению и как обработать поступившие данные
TCP/IP  - стек протоколов (более простой)
5) Сеансовый уровень - позволяет обеспечить сеанс связи и поддерживать долгосрочную организацию связи
6) Представления - отвечает за то, в какой форме представлять данные ASCII - код (формат приложения)
7) Прикладной

TCP/IP  - реализация с помощью 4-х уровней
1) Канальный (объединяет в себе физический и канальный (уровни L1 и L2), их реализация называется Ethernet)
2) Сетевой (уровень L3) - протокол IP
3) Транспортный - протоколы TCP и UDP (уровень L4)
4) Прикладной - уровень L7

^^8 .
/&*/---Компьютерные сети---/&*/Технология Ethernet/&*/* Для чего нужен механизм контрольной суммы в пакете /&*/

FSC - контрольная сумма(занимает 4 бита) - предназначена для защиты данных при передаче по сети
Отправитель при отправке пакета данные делит в нем по 4 бита, они в итоге складываются побитово и записывается результат в контрольную сумму и дальше отправляется по сети
Если на следующем этапе контрольная сумма не совпадёт (допустим в результате помехи исказятся данные и в результате сумма будет другая то такой пакет уничтожится и передаче не подлежит
как правило обычной суммой не ограничиваются а контрольные суммы вычисляются с помощью хэш-функции, которая позволяет приблизить к 0 возникновение хэш-коллизии

/&*/---Компьютерные сети---/&*/Технология Ethernet/&*/ Чем отличается устройство Hub от Switch /&*/
Устройство хаб при передаче одновременно информации третьему устройству создаст коллизию на неём (данные исказятся) и одновременно эти искаженныё данные поступят на 1 и 2 компьютер (возникнет домен коллизий)
Устройство свитч имеет встроенную память и позволяет разграничивать (буферизировать данные при одновременной передаче, запоминая кому передавались данные

^^12 .
/&*/---Компьютерные сети---/&*/Технология Ethernet/&*/ Что такое таблица MAC адресов комммутатора/&*/
MAC address table это таблиц MAC адресов коммутатора, при подключении сети она пустая, но быстро динамически заполняется
коммутатор запоминает на какой порт пришёл пакет от какого mac адреса и если ему еще не известно DST mac адрес на каком порте находится он рассылпает его
во все порты (unknown distanation unicast), кроме того, с которого он пришел, если оконечное устройство понимает что пакет пришел не ему, оно его гасит
при отключении устройства из порта кормутатора, в таблице mac адресов все данные, сохраненные за этим портом стираются

^^21 .
/&*/---Компьютерные сети---/&*/Сетевой уровень, протокол IP, уровень L3/&*/ как хранятся записи в таблице маршрутизации роутеров/&*/
записи в таблицах иаршрутизации роутеров хранятся в виде диапазонов ip адресов
диапазон ip адресов - ip сеть

192      .   168   .    123   .    10       //ip адрес
11000000  10101000   01111011   00001010
255      .   255   .    255   .    0        //ip маска (вначале идут только 1111 потом 0 )
11111111  11111111   11111111   00000000

Записав ip адрес хоста с маской мы можем вычислить адрес ip сети и диапазон в котором она лежит, просто побитово умножая каждый двоичный разряд сети на разряд маски

11000000  10101000   01111011   00000000
   192   .   168   .    123   .    0

Таким образом диапазон IP адресов для нашей IP сети будет выглядеть следующим образом: от 192.168.123.0 до 192.168.123.10
по-другому записывают 192.168.123.0/24    где 24 - количество единиц в маске сети
нельзя использовать 192.168.123.0   и   192.168.123.255  (служебные)

Что сделать, чтобы увеличить количество возможных ip адресов в сети

Необходимо в маске вместо последней единицы записать 0, теперь при значении соответсвующего разряда IP адреса (0 или 1) для каждого из вариантов будет по 253 варианта
диапазон будет выглядеть как от 192.168.122.0 до 192.168.123.255  только маска будет уже 255.255.254.0

^^23 .
/&*/---Компьютерные сети---/&*/Сетевой уровень, протокол IP, уровень L3/&*/ Что такое TTL (принцип работы)/&*/
TTL   - время жизни пакета (измеряется в количество "хопов" - количестве скачков между устройствами - после каждого перехода на один уменьшается
далее пакет уничтожается. Отправителю сообщается об уничтожении пакета. Это является встроенной защитой от запетлевания по аналогии с STP в уровне L2

Windows по умолчанию делает TTL 128, LINUX 64

^24 .
/&*/---Компьютерные сети---/&*/Сетевой уровень, протокол IP, уровень L3/&*/ Для чего нужна утилита tracert/&*/
утилита tracert  нужна, чтобы установить цепочку муршрутизации между хостами
допустим имее host 1 c ip адресом 1.1.1.1  и host 2 c ip адресом 8.8.8.8
Утилитой tracert создается пакет с srcIP 1.1.1.1   и dstIP 8.8.8.8  и TTL 1
Пакет прилетает на первый маршрутизатор TTL уменьшается на 1 и станоситься равно 0, пакет уничтожается и host 1 получает сообщение
о том, что время жизни пакета истекло и он уничтожен и адрес ip маршрутизатора на котором он был уничтожен, далее создаётся пакет
в котором TTL уже равно 2 , пакет уничтожается на втором маршрутизаторе и host 1 получает уже Ip адрес второго маршрутизатора
и так далее до тех пор, пока пакет не достигнет host 2, если же произойдет ошибка на каком либо участке, придут **** и можно будет судить
приблизительно после какого звена маршрутизации возникли неисправности

VLAN на коммуттаторе настраивается в режиме config  (в консоли коммутатора набираем последовательно команды ena И config t)

int fa0/1   // выбираем нужный интерфейс, на котором будем проводить настройку

switchport mode access	     // открываем порт в режиме acceess 

switchport access vlan 10   // помещаем порт в 10 порт влан

*******
чтобы задать имя влану в режиме config выполняем команды:

vlan 333    //влан с его номером, которому хотим присвоить имя

name TEST   // задание имени влану

далее выполняем следующий перечень команд:
	interface FastEthernet0/1     // настройка порта в режиме Access с вланом 10

^50 .
/&*/---Компьютерные сети---/&*/Сетевой уровень, протокол IP, Протоколы маршрутизации WLAN/&*/ Как настроить интерфейс (порт) в режиме Trunk с вланом 10 и 20/&*/

int gi0/1    //выбираем интерфейс коммутатора, который будет работать в режиме trunk 
switchport mode trunk   // переводит порт в режим транк (разрешает все вланы)

далее выполняем следующий перечень команд:
	interface GigabitEthernet0/1
	switchport mode trunk   // переводит порт в режим транк (разрешает все вланы)
	switchport trunk allowed vlan 10, 20  //разрешает вход трафика с вланов 10 и 20
	switchport trunk allowedi vlan add, remove) - добавить, удалить влан из списка

^51 .
/&*/---Компьютерные сети---/&*/Сетевой уровень, протокол IP, Протоколы маршрутизации WLAN/&*/ Как настроить VLAN на маршрутизаторе/&*/

допустим на интерфейсе gi0/0/0 роутера надо настроить два подинтерфейса влан 10 и влан 20
выполняем перечень команд:

	ena
	config t
	// Интерфейс, который идёт со свитча к роутеру должен быть настроен в режим trunk
	int gi0/0/0.20 
	encapsulation dot1Q   //указываем в каком влане будет находиться подинтерфейс
	ip address xx.xx.xx.xx  mm.mm.mm.mm  //ip адрес (тот адрес, который будет шлюзом по умолчанию в чужие сети)
	
выполняем следующий перечень команд в режиме конфигураций:
	interface GigabitEthernet0/0/0.10  //создание подинтерфейса на порту GigabitEthernet0/0/0 с ID 10 (как правило номер подинтерфейса равен номеру влана)
	encapsulation dot1Q   //указываем в каком влане будет находиться подинтерфейс
	ip address   //назначаем ip адрес на подинтерфейс
